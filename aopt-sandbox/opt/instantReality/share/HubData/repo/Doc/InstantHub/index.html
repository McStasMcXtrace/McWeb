instantHub REST API Description

  • / - Frontpage

  • /index - Frontpage

  • /hub - instantHub self description

    • allowed http verbs: GET, HEAD
    • default output: HTML
    • alternative output json (Accept: application/json)

  • /service - instantService registry

    • GET - retrieve registeres service list

      • query parameter: class, only list services of type class
      • default output: HTML
      • alternative output json (Accept: application/json)

    • POST - register instantService

  • /job - job registry

    • GET - retrieve job list

      • query parameter: class, only list services of type class
      • default output: HTML
      • alternative output json (Accept: application/json)

    • POST - add a job

      • POST request - expects a JSON file describing the job service class and job parameters
      • The POST request JSON block follows the template below 
                      {
                "ServiceClass" : "<Requested ServiceClass>"
                "Parameters"   :
                {
                  "<parameter name 0>" : "<parameter value 0>",
                  "<parameter name 1>" : "<parameter value 1>",
                  .
                  .
                  .
                  "<parameter name N>" : "<parameter value N>",
                }
              }
      • Return value:
        • Default: the URI of the result
        • The return value might be modified by the reply parameter (service class dependent)

  • /job/[jobid] - job details

    • GET - retrieve job details

      • default output: HTML
      • alternative output json (Accept: application/json)

    • DELETE - delete a job

  • /job/updateState - batch job state update

    • POST - retrieve job details

      • default output json
